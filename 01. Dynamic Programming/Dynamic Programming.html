Dynamic Programming¶

In dynamic programming, We will solve problems that that are slightly easier than the reinforcement learning problem. More specifically:

  • Instead of working in a setting where the agent has to learn from interaction, We'll assume that the agent already knows everything about the environment. So it knows how the environment decides the next state, And it also knows how the environment decides reward.
  • The goal is to find the optimal policy

# Iterative Policy Evaluation Algorithms

def policy_evaluation(env, policy, gamma=1, theta=1e-8):
    """
    implementation of iterative policy evaluation.
    
    ARGUMENTS
    -----------------------------
        - env: This is an instance of an OpenAI Gym environment, where env.P returns the one-step dynamics.
        - policy: This is a 2D numpy array with policy.shape[0] equal to the number of states (env.nS), and 
                  policy.shape[1] equal to the number of actions (env.nA).  policy[s][a] returns the probability 
                  that the agent takes action a while in state s under the policy.
        - gamma: This is the discount rate. It must be a value between 0 and 1, inclusive (default value: 1).
        - theta:  This is a very small positive number that is used to decide if the estimate has sufficiently 
                  converged to the true value function (default value: 1e-8).
    
    RETURNS
    -----------------------------
        - V: This is a 1D numpy array with V.shape[0] equal to the number of states (env.nS). V[s] contains the 
             estimated value of state s under the input policy.
    
    """
    # Initializing the state-value function with zeros
    V = np.zeros(env.nS)
    
    # Inifite loop
    while True:
        
        # Initializing the delta (amount of value change at each iteration) with zero
        delta = 0
        
        # Iterating through states
        for s in range(env.nS):
            
            # Initialize the value of current state with zero
            Vs = 0
            
            # Iterate through actions and their probabilities
            for a, action_prob in enumerate(policy[s]):
                
                # Iterate through all the one-step dynamics of enviroment
                for prob, next_state, reward, done in env.P[s][a]:
                    
                    # Update the value of current state 
                    Vs += action_prob * prob * (reward + gamma * V[next_state])
            
            # Updating the delta (amount of value change at each iteration)
            delta = max(delta, np.abs(V[s]-Vs))
            
            # Append the value of current state to the state-value function
            V[s] = Vs
            
        # Break if delta (amount of value change at each iteration) is less than theta (small positive number that is used to decide if the estimate has sufficiently converged to the true value function)
        if delta < theta:
            break
            
    return V

def q_from_v(env, V, s, gamma=1):
    """
    Function for converting state-value function v to action-value function q
    
    ARGUMENTS
    ------------------
        - env: This is an instance of an OpenAI Gym environment, where env.P returns the one-step dynamics.
        - V: This is a 1D numpy array with V.shape[0] equal to the number of states (env.nS). V[s] contains 
             the estimated value of state s.
        - s: This is an integer corresponding to a state in the environment.  It should be a value between 0
             and (env.nS)-1, inclusive.
        - gamma: This is the discount rate. It must be a value between 0 and 1, inclusive (default value: 1).
    
    RETURNS
    ------------------
        - q: This is a 1D numpy array with q.shape[0] equal to the number of actions (env.nA). q[a] contains 
             the (estimated) value of state s and action a.
    
    """
    # Initialize the value of actions with zeros
    q = np.zeros(env.nA)
    
    # Iterate through all actions
    for a in range(env.nA):
        
        # Iterate throught the one-step dynamics of enviroment
        for prob, next_state, reward, done in env.P[s][a]:
            
            # Updatet the value of each action
            q[a] += prob * (reward + gamma * V[next_state])
            
    return q

4. Policy Improvement¶

In this section, you will write your own implementation of policy improvement.

We use iterative policy evaluation to get the state-value function of a given policy. And we use policy improvement to get a new policy that's at least as good as the current one. So it's a good idea to loop over iterative policy evaluation and policy improvement until we converge to an optimal policy.

For finding a policy that is at least as good as the current one, We need to divide the policy improvement into 2 parts:

  • Convert the state-value function into the action-value function
  • Find the optimal policy based on the action-value function

The algorithm for policy improvement is as follows:

# Function for policy improvement
def policy_improvement(env, V, gamma=1):
    """    
    Policy Improvement algorithm for getting a new policy that's at least as good as the current one.

    ARGUMENTS
    -------------------
        - env: This is an instance of an OpenAI Gym environment, where env.P returns the one-step dynamics.
        - V: This is a 1D numpy array with V.shape[0] equal to the number of states (env.nS). V[s] contains 
             the estimated value of state s.
        - gamma: This is the discount rate. It must be a value between 0 and 1, inclusive (default value: 1).


    RETURNS
    -------------------
        - policy: This is a 2D numpy array with policy.shape[0] equal to the number of states (env.nS), and 
                  policy.shape[1] equal to the number of actions (env.nA).  policy[s][a] returns the probability 
                  that the agent takes action a while in state s under the policy.
    """
    # Initialize an equal probable random policy
    policy = np.zeros([env.nS, env.nA]) / env.nA
    
    # Iterate through states
    for s in range(env.nS):
        
        # Get the action-value function
        q = q_from_v(env, V, s, gamma)
        
        # OPTION 1: Construct a deterministic policy 
        # policy[s][np.argmax(q)] = 1
        
        # OPTION 2: Construct a stochastic policy that puts equal probability on maximizing actions
        best_a = np.argwhere(q==np.max(q)).flatten()
        policy[s] = np.sum([np.eye(env.nA)[i] for i in best_a], axis=0)/len(best_a)
        
    return policy

5. Policy Iteration¶

So far we used policy evaluation to determine how good a policy is by calculating its state-value function. We also used policy improvement which uses the state-value function for a policy to construct a new policy that's better or equal to the current one. It makes sense to combine these two algorithms to produce an algorithm that successively proposes better and better policies. The name for algorithm that combines these two steps is called policy iteration. In this section, you will write your own implementation of policy iteration.

You can find the pseudocode for policy iteration in below:

import copy

def policy_iteration(env, gamma=1, theta=1e-8):
    """
    Combining the policy evaluation and policy improvement algorithm and looping over them.
    
    ARGUMENTS
    --------------------
        - env: This is an instance of an OpenAI Gym environment, where env.P returns the one-step dynamics.
        - gamma: This is the discount rate. It must be a value between 0 and 1, inclusive (default value: 1).
        - theta: This is a very small positive number that is used to decide if the policy evaluation step has 
                 sufficiently converged to the true value function (default value: 1e-8).

    RETURNS
    --------------------
        - policy: This is a 2D numpy array with policy.shape[0] equal to the number of states (env.nS), and policy.
                  shape[1] equal to the number of actions (env.nA).  policy[s][a] returns the probability that the 
                  agent takes action a while in state s under the policy.
        - V: This is a 1D numpy array with V.shape[0] equal to the number of states (env.nS).  V[s] contains the 
             estimated value of state s.
    """
    # Initial guess for our policy; we use equal probable random policy (for each state, each action is equally likely to be chosen)
    policy = np.ones([env.nS, env.nA]) / env.nA
    
    # Infinite loop
    while True:
        
        # Calcuate the policy evaluation
        V = policy_evaluation(env, policy, gamma, theta)
        
        # Calculate the policy improvement
        new_policy = policy_improvement(env, V)
        
        # OPTION 1: Stop if the policy is unchanged after an improvement step
        if (new_policy == policy).all():
            break;
        
        # OPTION 2: Stop if the value function estimates for successive policies has converged
        # if np.max(abs(policy_evaluation(env, policy) - policy_evaluation(env, new_policy))) < theta*1e2:
        #    break;
        
        # Update the policy
        policy = copy.copy(new_policy)
        
    return policy, V

6. Truncated Policy Iteration¶

In this section we will look at some other variation of iterative policy iteration algorithm. If you remember in iterative policy iteration algorithm, we would break the look if the value change was smaller than theta. However in truncated policy evaluation instead of terminating the algorithm with stopping criteria, we will give an absolute number of iterations that we're willing to calculate.

In this section, you will write your own implementation of truncated policy iteration.

def truncated_policy_evaluation(env, policy, V, max_it=1, gamma=1):
    """
    Truncated Policy Evaluation algorithm.
    
    ARGUMENTS
    --------------------
        - env: This is an instance of an OpenAI Gym environment, where env.P returns the one-step dynamics.
        - policy: This is a 2D numpy array with policy.shape[0] equal to the number of states (env.nS), and 
                  policy.shape[1] equal to the number of actions (env.nA).  policy[s][a] returns the probability 
                  that the agent takes action a while in state s under the policy.
        - V: This is a 1D numpy array with V.shape[0] equal to the number of states (env.nS).  V[s] contains the 
             estimated value of state s.
        - max_it: This is a positive integer that corresponds to the number of sweeps through the state space 
                  (default value: 1).
        - gamma: This is the discount rate. It must be a value between 0 and 1, inclusive (default value: 1).
    
    RETURNS
    --------------------
        - V: This is a 1D numpy array with V.shape[0] equal to the number of states (env.nS). V[s] contains the 
             estimated value of state s.
    """
    # Initializing the number of iterations to zero
    num_it=0
    
    # Loop max_it number of time
    while num_it < max_it:
        
        # Iterate through states
        for s in range(env.nS):
            
            # Initialize the value of current state with zero
            v = 0
            
            # Get the action-value of current state
            q = q_from_v(env, V, s, gamma)
            
            # Iterate through all possible actions of current state
            for a, action_prob in enumerate(policy[s]):
                
                # Calculate the value of current state
                v += action_prob * q[a]
                
            # Append the value of current state to the state-value function
            V[s] = v
            
        # Increment the number of iterateions
        num_it += 1
        
    return V

def truncated_policy_iteration(env, max_it=1, gamma=1, theta=1e-8):
    """
    Truncated Policy Iteration algorithm.
    
    ARGUMENTS
    ---------------------
        - env: This is an instance of an OpenAI Gym environment, where env.P returns the one-step dynamics.
        - max_it: This is a positive integer that corresponds to the number of sweeps through the state space
                  (default value: 1).
        - gamma: This is the discount rate. It must be a value between 0 and 1, inclusive (default value: 1).
        - theta: This is a very small positive number that is used for the stopping criterion (default value: 1e-8)
    

    RETURNS
    ---------------------
        - policy: This is a 2D numpy array with policy.shape[0] equal to the number of states (env.nS), and 
                  policy.shape[1] equal to the number of actions (env.nA).  policy[s][a] returns the probability 
                  that the agent takes action a while in state s under the policy.
        - V: This is a 1D numpy array with V.shape[0] equal to the number of states (env.nS). V[s] contains the 
             estimated value of state s.
    """
    # Initializ the state-value function with zeros
    V = np.zeros(env.nS)
    
    # Initializ the policy with zeros
    policy = np.zeros([env.nS, env.nA]) / env.nA
    
    # Infinite loop
    while True:
        
        # Calculate the policy
        policy = policy_improvement(env, V)
        
        # Get a copy of state-value function
        old_V = copy.copy(V)
        
        # Calculate the truncated policy evaluation
        V = truncated_policy_evaluation(env, policy, V, max_it, gamma)
        
        # If the amount of change was less than theta then break
        if max(abs(V-old_V)) < theta:
            break
            
    return policy, V

7. Value Iteration¶

So far we talked about the policy iteration and truncated policy iteration which in case of the second one, the policy evaluation step is only permitted a limited number of sweeps through the state space. In other word, we limit the number of times that the estimated value of each state is updated before proceeding to the policy improvement round.

In value iteration, the policy evaluation step is stopped after a single sweep. As simple as that.

def value_iteration(env, gamma=1, theta=1e-8):
    """
    Value Iteration algorithm.
    
    ARGUMENTS
    -----------------
        - env: This is an instance of an OpenAI Gym environment, where env.P returns the one-step dynamics.
        - gamma: This is the discount rate. It must be a value between 0 and 1, inclusive (default value: 1).
        - theta: This is a very small positive number that is used for the stopping criterion (default value: 1e-8).
    
    RETURNS
    -----------------
        - policy: This is a 2D numpy array with policy.shape[0] equal to the number of states (env.nS), and policy.
                  shape[1] equal to the number of actions (env.nA).  policy[s][a] returns the probability that the 
                  agent takes action a while in state s under the policy.
        - V: This is a 1D numpy array with V.shape[0] equal to the number of states (env.nS).  V[s] contains the 
             estimated value of state s.
    """
    # Initializ the state-value function with zeros
    V = np.zeros(env.nS)
    
    # Infinite look
    while True:
        
        # Initialize delta with zero
        delta = 0
        
        # Iterate through states
        for s in range(env.nS):
            
            # Get the value of current state
            v = V[s]
            
            # Update the state-value function
            V[s] = max(q_from_v(env, V, s, gamma))
            
            # Update the delta
            delta = max(delta, abs(V[s]-v))
            
        # Break the loop if theta bigger than delta
        if delta < theta:
            break
            
    # Apply the policy improvement step
    policy = policy_improvement(env, V, gamma)
    
    return policy, V
